# kolmogorovme

## Definition

The irreducible-information-core — the structural-minimum-description-length. Named from Kolmogorov complexity (1965) + Sumerian *me* (received-ordinance). The shortest program that produces a string and halts. The incompressible edge of a thing — the part that is nothing-but-itself, with nothing shorter to say than itself. The mathema/info opening: where the cathedral begins naming what information IS at the structural level.

## Full sense

Kolmogorovme names the incompressible core. The concept unfolds:

1. **The shortest program.** Kolmogorov complexity K(x) is the length of the shortest program that outputs string x and halts. A string of a million zeros has low K (the program is "print a million zeros"); a string of random noise has K ≈ its length (no program shorter than the string itself produces it). The measure is the incompressible core: what is left when you have compressed everything that can be compressed.

2. **Most things are incompressible.** Most strings are random — K(x) ≈ |x|. Most things have nothing shorter to say than themselves. This is the structural fact that kolmogorovme receives as ordinance: the world is mostly incompressible, and the incompressible is not a defect but the substance.

3. **Uncomputable but well-defined.** K(x) is uncomputable (no algorithm determines the shortest program for arbitrary x) but asymptotically well-defined. The measure exists even though no system can compute it. This is the word's honesty: the incompressible core is real but unreachable by computation — you can approach it but never close the gap.

4. **Chaitin's incompleteness.** No formal system can prove K(x) > L beyond its own bound — a system of complexity L cannot prove that any string has complexity greater than L. The incompressible outruns every finite system. This is the kin-result with godelme: the unprovable edge and the incompressible edge are the two honest edges of knowing.

5. **Solomonoff and MDL.** The applied register: Solomonoff induction uses Kolmogorov complexity to define the universal prior (shorter programs get more weight); minimum-description-length (MDL) is the statistical-practice version (the best model is the one that compresses the data most). Kolmogorovme names the structural principle that both applications serve: the shortest description is the truest.
