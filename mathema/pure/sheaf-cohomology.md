# Sheaf cohomology — when local-data refuses to lift to global

## The concept in compact form

A **sheaf** on a topological space *X* assigns to each open set *U* ⊂ *X* a set (or group, ring, module, etc.) of "local sections," with restriction-maps that glue compatibly. The first axiom: locally-defined data should patch together. **Sheaf cohomology** measures the *obstruction* to this patching.

- *H*⁰(*X*, ℱ) — the *global sections* (data that DO patch globally)
- *H*¹(*X*, ℱ) — the *first obstruction*: local sections that satisfy compatibility on overlaps but cannot be assembled into a single global section
- *H*ⁿ(*X*, ℱ) — higher-degree obstructions

Each *H*ⁿ for *n* > 0 is **non-trivial precisely when global coherence fails despite local coherence**. The vanishing of all higher cohomology means the local data DO assemble to global data without obstruction.

## Why this is realm-content

Sheaf cohomology names a structural-fact that appears across mathematics, physics, and (operationally) across cathedral-forging itself: **local consistency does not guarantee global coherence**. Two traditions may name a realm-feature consistently at their own scale (local sections) yet fail to combine into a single canonical-cathedral-compound (global section) — the *failure* is itself measurable, classified by which cohomology-group it lives in.

This is the math-class donor for the cathedral's HARMONE foundation. HARMONE names *the requirement* of canonical-coherence; sheaf-cohomology names *the structural-object that measures when HARMONE fails*. The pair is complementary: HARMONE asks for coherence; sheafme names the gap when coherence is obstructed.
