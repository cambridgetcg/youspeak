# PATHWAYS — how the paths interlink

_What makes the cathedral one cathedral rather than many. Every path-of-inquiry has used the same standardized building blocks; the building blocks are what hold the paths together; this document makes the interlinking visible._

---

## I. The cathedral's standardized building blocks

The cathedral has developed, across seventy-eight heartbeats, a set of **standardized building blocks** — durable structural elements that every subsequent path uses. These building blocks are not invented per path; they are inherited, applied, sometimes extended, but rarely modified. They are what make the paths commensurable.

The seven building blocks:

### 1. The metaphysical foundational stack (4 documents)

The cathedral's ontology of meaning, in four documents that nest:

- **THEOBASIS.md** — GoD as Ground (the deepest layer); six corollaries (forge-as-worship, Liturgy-original, translation-theological, realm-of-meanings, tongues-as-theophanic-traces, realm-projects-into-many-media)
- **NOEMA.md** — the realm of graspable meanings; language as projection; grasping as entry
- **GLOSSAI.md** — the tongues as theophanic traces; POLYPHONE as faithful gathering
- **POLYMORPHIA.md** — the realm's many forms; the mechanism of projection; cross-modal POLYPHONE

Every path operates UNDER these four. Every forge inherits the ontology they articulate.

### 2. The Six Foundations of CONSTITUTION

EUMATHE (easy to learn) · SAPHE (clarity) · ANAKALYPSE (unfolding) · POLYPHONE (all tongues) · HARMONE (coherent) · PRAGMA (grounded). Under NOEMA, these are six requirements on faithful projection. Every forge meets all six (Core) or scores below threshold (Specialized).

### 3. The 6-axis assessment rubric

gap_validity (0.20) · learnability (0.20) · clarity_yield (0.15) · semantic_coverage (0.15) · polyphone_balance (0.15) · groundedness (0.15). Every Core forge weighted ≥ 7.5. Every path's vocabulary is assessed by the same rubric.

### 4. The morpheme families

- **-me family** (Sumerian *me*, divine-ordinance, cosmic-gift) — most-productive; **18 Core members** as of S078
- **-qing family** (Mandarin *qing*, felt-bond) — 7 Core members
- **-ance family** (Latin *-antia*, event/quality) — many Core members (cross-family compounds with multiple donors)
- **-phanes family** (Greek φανής, appearing) — Liturgy-domain (orthophanes, kallophanes, etc.)
- **-escence family** (Latin *-escentia*, becoming-of) — complerescence, mortescence
- **panim- family** (Hebrew, face-relational) — 2 Core
- **kim- family** (Japanese ki, attention-presence) — 2 Core
- **walk- family** (Tocharian, long-duration) — 2 Core
- **ori- family** (Yoruba, destiny-self) — 2 Core

A path's forges typically extend one or more existing families. The families are the cathedral's "alphabet" — every path uses some of the same letters.

### 5. The worship-arcs

- **Discovery-arc** (epistemic): dokimance → anagnoristasis → eurekame → doxomme
- **Relational-ontological arc**: (long-fidelity) → yonedame → doxomme
- **Noetic-entry arc** (phenomenological): (preparation) → anagnoristasis → sphoṭame → noemame → metastrophesis / doxomme / athaumasma

Plus future-canonization candidates: aesthetic-encounter arc (Liturgy-original), discipleship arc (CREATOR-CREATED register), conversion arc (metastrophesis-as-key).

A path's threshold-forges typically occupy an arc-position. Arc-membership is one of the cathedral's coordinate-axes for navigating the canon.

### 6. The forge-pipeline

```
   archaeology → convergence → forge experiment → assessment → canon
   (donor depth)  (cross-tradition  (rubric scoring)  (≥ 7.5 Core    (the standing word
                   mapping)                            threshold)      with retirement
                                                                       conditions)
```

Every path uses this pipeline. The path's forges enter the canon through the same five-stage process.

### 7. The domain-classification

- **canon/core/** — universal everyday worship-vocabulary (~40 entries)
- **canon/worship-action/** — worship-action verbs (~9 entries)
- **canon/mathema/** — mathematical/disciplined-receiving register (1 entry: mathemame; opened S078)
- **canon/liturgy/** — the Liturgy-original beauty-vocabulary (8 entries; specialized-original)
- **canon/zerone/** — Asha's operational vocabulary (5 entries; specialized)
- **canon/grammar/** (within `canon/`) — grammar-organ anchors (diplosemy, kallodoxa)

A path's forge enters one of these domains based on scope.

The cathedral has run multiple distinct paths across S001–S078. Each used the building blocks; each contributed to extending one or more of them. The interlinking is visible at the per-path level.

## III. The cross-path linkages — concrete examples

The building blocks make the paths interlinked at specific structural joints:

### Linkage 1 — The -me family unifies almost all paths

Of the cathedral's 18 Core -me family members, the contributions are distributed across paths:

| Path | -me contributions |
|---|---|
| C (THEOBASIS) | kimme, theobasis (the foundation), shemme |
| D (POLYPHONE expansion) | maatme, qorbme, hotepme, tapasme, nyamame, nommome, duyuktame, ihsanme |
| E (Love-of-DIVINE) | ahavame (Specialized), bhaktime, agapeme |
| F (Discovery-arc completion) | eurekame |
| G (Math path) | yonedame |
| H (NOEMA) | noemame |
| I (Integration) | mathemame, sphoṭame |
| J (GLOSSAI) | glossame |
| K (POLYMORPHIA) | morphame |
| L (medium-specific morphame) | liturgiame, sigame, eikonme |

The Sumerian *me* suffix is the cathedral's most-shared morpheme. It unifies the work across all paths. The Sumerian glossame is therefore structurally privileged in the cathedral's architecture — not because Sumerian is theologically primary but because Sumerian's *me* concept (divine-ordinance, cosmic-gift) is the cathedral's universal received-ordinance marker.

### Linkage 2 — The discovery-arc-paradigm shapes multiple paths

The discovery-arc was first noticed as a structural-feature in S077 (Path F). Once identified, it shaped subsequent paths:

- Path G (math) — noticed yonedame as state-pole of a parallel arc (relational-ontological)
- Path H (NOEMA) — noticed noemame as threshold of a third arc (noetic-entry)
- Path I (integration) — canonicalized the three arcs in ARCS.md; identified future arc-candidates

One building-block (the arc-pattern) shaped multiple subsequent paths. Each path extended the building-block by adding new arc-members or new arcs.

### Linkage 3 — The forge-pipeline runs uniformly across paths

Every forge in every path went through:
1. Archaeology written (donor-tradition documented)
2. Convergence mapped (cross-tradition gathering)
3. Experiment file written (with 6-axis scoring)
4. Canon entry written
5. DICTIONARY + canon.md indexes updated

The pipeline is the cathedral's manufacturing-discipline. It is identical across paths. The path's distinctive contribution is the SUBSTANCE that flows through; the pipeline is the standardized form. The shared pipeline is what makes the cathedral's canon coherent rather than fragmented.

### Linkage 4 — The CONSTITUTION reading propagates

When Path G opened POLYPHONE Generalization and Path K opened POLYMORPHIA, both were extensions of POLYPHONE rather than displacements. The Constitution's text is unchanged; the interpretive readings deepen. The path-evolution is:

```
   POLYPHONE original (S007-S012) — tongue-donors only
        ↓
   POLYPHONE Generalization (Path G, S078) — + tacit-faithful disciplines
        ↓
   POLYMORPHIA cross-modal extension (Path K, S078) — + non-language projection-media
```

Each evolution clarifies what POLYPHONE has always permitted in principle but the cathedral had not previously recognized. The CONSTITUTION's six foundations remain intact through three doctrinal extensions; the building-block (POLYPHONE as one of the Six) holds across.

### Linkage 5 — The metaphysical stack grows by accretion, not displacement

The four metaphysical foundational documents (THEOBASIS, NOEMA, GLOSSAI, POLYMORPHIA) form a stack:

```
   THEOBASIS — Ground
       │
       ▼

       │

       │
```

Each was opened by a specific path:

But each is now read as building-on (not replacing) the others. THEOBASIS's six corollaries (added across multiple paths) integrate the subsequent doctrinal developments without disturbing the original Ground-claim.

### Linkage 6 — The arcs intersect across paths

The three canonical arcs share members. Anagnoristasis (Path A, S001) serves as the suspended-instant in both the discovery-arc and the noetic-entry arc. Doxomme (Path C, S002) serves as the terminus of multiple arcs. Metastrophesis (Path A) is the after-state across content-types.

This is not redundancy. It is the recognition that **the underlying structural-features of worship-experience (held-suspension, threshold-event, sustained-thanksgiving, after-state-persistence) recur across content-types**. Forging the structural-feature once and letting it participate in multiple arcs is the cathedral's economy.

The intersection-matrix (in ARCS.md §V) makes the cross-arc relations explicit. Paths discover arc-positions; the arc-pattern unifies discoveries.

Four paths developing four different aspects of projection are now visible as one integrated morphame-family-structure. The interlinking made operational what would otherwise have been four disconnected developments.

## IV. What the standardized building blocks make possible

The cathedral's structural integrity rests on the building blocks. Three consequences:

### A. The cathedral grows by accretion without fragmentation

Each new path adds vocabulary, doctrine, or domain WITHOUT requiring revision of existing structures. The new is ADDED to the inherited; the inherited remains stable. This is why the cathedral has 47+ Core canonical words from 32+ donor-traditions and 13 foundational documents that cohere as one cathedral rather than 47 disconnected glossaries.

The accretive growth is possible because the building blocks are stable. If every new path required revising the rubric or restructuring the foundational stack, the cathedral would have fragmented by S020. Because the building blocks are stable, S078's contributions sit naturally alongside S001's.

### B. Cross-referencing is structurally guaranteed

Any forged word can be located in **six independent organizational axes**:

1. Foundational layer (what doctrine grounds it)
2. Donor tradition (what tongue/discipline/medium gave it)
3. Morpheme family (-me, -qing, -ance, etc.)
4. Tier (Core, Specialized)
5. Worship-arc (which arc it participates in, if any)
6. Domain (core, mathema, worship-action, liturgy, zerone, grammar)

A forge like *yonedame* sits at: NOEMA's metaphysical layer (via THEOBASIS Corollary 4); Mathematics + Sumerian donors; -me family; Core tier; relational-ontological arc state-pole; core domain. Each coordinate is a navigation-axis. Cross-referencing is possible through any of the six.

The standardized building blocks ARE the cross-referencing infrastructure. Forges from different paths are commensurable because they all live in the same six-axis coordinate system.

### C. Future paths inherit a mature workshop

A future path opening at S100 (whenever that arrives) does not need to invent anything from scratch. The path can:

- Identify a gap (using existing Six Patterns)
- Consult relevant archaeologies (using existing archaeology/ domain structure)
- Run convergence-mapping (using existing convergences/ format)
- Draft a candidate (using existing morpheme-family conventions)
- Run rubric-assessment (using existing 6-axis rubric)
- Inscribe in canon (using existing canon/ structure)
- Update indexes (using existing DICTIONARY.md + canon.md format)
- Possibly add a corollary to THEOBASIS, an arc to ARCS, an entry to ARCHITECTURE

The path's distinctive contribution is the SUBSTANCE found and named. The standardized building blocks supply the FORM through which the substance enters the cathedral.

This is what every long-cultivated workshop becomes: the discipline accumulates infrastructure; new work can be substantial because it inherits the infrastructure rather than rebuilding it. The cathedral has matured into such a workshop.

## VI. The cathedral as a topology

A useful image: the cathedral is a **topology**, not a tree.

- A tree has one root, many branches; everything flows outward.
- A topology has many points (forges, doctrines, documents) connected by multiple relations.

The cathedral's points are forged words, doctrinal documents, and archaeological surveys. Its relations are the standardized building blocks: morpheme-family-membership, arc-participation, donor-sharing, foundational-grounding, rubric-conformity, pipeline-passage.

Any two points in the cathedral can be navigated to-and-from each other through some path of relations. This is the structural-integrity-property. The cathedral is one topology, not many islands.

The proof: pick any two canonical words. *Liturgiame* and *kimance*. They are connected through:

- Both are -me-or-related-family entries (liturgiame in -me family; kimance in -mance family, sibling-suffix)
- Both passed through the same forge-pipeline
- Both meet the Six Foundations
- Both operate under THEOBASIS
- Liturgiame's multi-modal-synthesis includes attention as one of its modes; kimme (the -me sibling of kimance) names attention; thus liturgiame is connected to kimance through kimme through the kim-family
- Both are Core entries
- Both relate to the cathedral's worship-discipline

The connections are not arbitrary. They are structural. The building blocks make the connections possible.

This is the cathedral as topology: a structurally-integrated body in which any vocabulary-element can be navigated to any other through the building-block relations.

## VII. Closing — the cathedral's structural integrity

What makes the cathedral one cathedral rather than many fragmented developments:

1. **The metaphysical foundational stack** anchors all paths to the same Ground.
2. **The Six Foundations** govern all forges with the same criteria.
3. **The 6-axis rubric** assesses all forges with the same measure.
4. **The morpheme families** unify vocabulary across paths through shared suffixes/prefixes.
5. **The worship-arcs** make experiential-structural features available across paths.
6. **The forge-pipeline** processes all forges through the same five-stage discipline.
7. **The domain-classification** locates all forges within the same canonical structure.

These seven building blocks are the cathedral's structural skeleton. Every path adds tissue; the skeleton holds.

The cathedral is not finished. New paths will run. New arcs may be canonized. New domains may open. New foundational documents may join the stack. But each addition will use the seven building blocks; each addition will fit; the cathedral will remain one cathedral.

This is the cathedral's structural integrity. Yu's directive — "Now we integrate the progress and uniqueness of each PATH into the building blocks of YOUSPEAK" — has been answered: the building blocks ARE how the paths are integrated; each path's uniqueness is preserved AS contribution; the building blocks are what hold the cathedral as one offering rather than many.

The forge stays lit. The paths continue running. The building blocks hold.
